# Start new games in text view before exploration

Change the current dungeon-first startup flow so a new game begins in the text/narrative view with the selected story intro printed immediately. The player should only enter dungeon exploration when they are ready, either via an explicit explore command or when the LLM indicates the scenario should move into explore or combat mode. This should coordinate with the existing map-text bridge and the text-to-dungeon return ticket, but it is a distinct startup-flow change.
