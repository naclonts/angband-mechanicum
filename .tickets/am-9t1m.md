# Combat: control all party members from story mode

In combat mode, the player should be able to control all current party members, not just the Tech-Priest. The combat roster should be derived from the current narrative game state — pull party members from the entity registry (characters tagged as allies/party). If a party member died or was separated in story mode, they should not appear in combat. Each party member needs combat stats (derived from their narrative description/role — e.g. Skitarius Alpha-7 is ranged, Enginseer Volta is melee). Turn structure becomes: player moves/acts with each party unit, then enemies act.
