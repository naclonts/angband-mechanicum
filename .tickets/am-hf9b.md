# Make story starts drive dungeon environments, factions, and set pieces

Story starts currently enter dungeon exploration through a thin environment inference layer, which is too weak to preserve scenario-specific biome, faction, and landmark identity. Build a full story-to-dungeon integration system so each story start can declare or resolve the dungeon environment, expected hostile factions, allied/neutral factions, landmark/set-piece requirements, and content exclusions. Example: the titan recovery start in the ash wastes should yield ash-waste/titan-graveyard spaces with ork lootas, scavengers, wreckage, titan internals, and possible ork fortifications, not generic forge contacts and terminals. Update docs/tests and preserve save/load continuity.

**2026-03-29T19:34:31Z**

Refinement from design review:\n- Treat environment as canonical session state shared by both text mode and dungeon mode, not as a dungeon-only inference. Add a current_environment_id (or equivalent structured location profile key) to engine/app/session state so narrative, travel, and dungeon generation all reference the same environment identity.\n- Story starts should seed the initial environment explicitly, but travel must be able to change it later. Example: start in ash wastes, then travel to a forge world; text mode should update the canonical environment/profile before the next dungeon session is generated.\n- Use a hybrid resolution model for travel and scene changes: deterministic matching first when the request cleanly maps to known environments/locations; optional LLM guidance when the destination is fuzzy, metaphorical, or compound; deterministic fallback if the LLM result is missing/invalid. The final persisted environment id must always be one of the supported canonical environment/profile ids.\n- Track both coarse environment and richer scenario/location profile where useful. Environment id can drive terrain palette and broad generation; a higher-level location/story profile can add faction, landmark, and exclusion rules. This shared state should be available to text prompts in the future as well as dungeon generation now.\n- Update the ticket implementation plan accordingly: environment transitions are not only a story-start problem, but a unified text<->dungeon world-state problem.
