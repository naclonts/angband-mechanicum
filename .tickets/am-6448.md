# Dungeon level data model

Persistent tile map data model for exploration-scale dungeons. Extends or replaces the existing combat Grid (24x17) with large multi-room dungeon floors (80x50+). Must support: terrain types (floor, wall, door, stairs-up, stairs-down, water, lava, etc.), item placement on tiles, NPC/creature placement, fog-of-war state (hidden/explored/visible), and serialization for save/load. The existing combat_engine.Grid can be extended or a new DungeonLevel class created. Key design: this is for exploration, not just combat arenas. Reference: TOME t-engine4 map system.
